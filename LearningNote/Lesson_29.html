CSS Reset and Normalize

1. CSS Reset

CSS reset is a collection of CSS rules intended to strip the default styling provided by browsers. The goal is to create a level playing field across different browsers, ensuring that all elements have no default margins, paddings, or other styles that could affect the consistency of your design.
Example: 
/* CSS Reset Example */
html, body, div, span, applet, object, iframe,
h1, h2, h3, h4, h5, h6, p, blockquote, pre,
a, abbr, acronym, address, big, cite, code,
del, dfn, em, img, ins, kbd, q, s, samp,
small, strike, strong, sub, sup, tt, var,
b, u, i, center,
dl, dt, dd, ol, ul, li,
fieldset, form, label, legend,
table, caption, tbody, tfoot, thead, tr, th, td,
article, aside, canvas, details, embed,
figure, figcaption, footer, header, hgroup,
menu, nav, output, ruby, section, summary,
time, mark, audio, video {
    margin: 0;
    padding: 0;
    border: 0;
    font-size: 100%;
    font: inherit;
    vertical-align: baseline;
}

2. Normalize.css

Normalize.css is a modern alternative to CSS reset. It makes browsers render all elements more consistently and in line with modern standards. It also corrects bugs and common browser inconsistencies.
Example: 
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/normalize/8.0.1/normalize.min.css">

3. Differences Between Reset and Normalize

  • CSS Reset removes all default browser styling. It sets elements to have no margin, padding, or other styles, creating a blank slate.
  • Normalize.css preserves useful default styles while correcting common issues. It provides a baseline for browser consistency without removing all styles.

4. Custom Reset Styles

Custom reset styles can be created to fit specific needs, combining the benefits of both CSS reset and Normalize.css.
Example: 
/* Custom Reset Styles */
* {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
}
body {
    line-height: 1.5;
    font-family: Arial, sans-serif;
}

5. Browser Default Styles

Browser default styles are the styles that browsers apply to HTML elements if no CSS is provided. These can vary between browsers and often need to be overridden for consistent design.

Example Code 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CSS Reset and Normalize</title>
    
    <!-- Normalize.css -->
    <link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/normalize/8.0.1/normalize.min.css">
    
    <!-- Custom Reset Styles -->
    <style>
        * {
            margin: 0;
            padding: 0;
            box-sizing: border-box;
        }
        body {
            line-height: 1.5;
            font-family: Arial, sans-serif;
        }
        /* Example content styling */
        .container {
            max-width: 800px;
            margin: 50px auto;
            padding: 20px;
            background-color: #f4f4f4;
            border: 1px solid #ddd;
            border-radius: 8px;
        }
        h1, h2, p {
            margin-bottom: 16px;
        }
    </style>
</head>
<body>
    <div class="container">
        <h1>CSS Reset and Normalize</h1>
        <h2>1. CSS Reset</h2>
        <p>A CSS reset strips all default browser styles, creating a blank slate for styling.</p>

        <p>Normalize.css makes browsers render all elements more consistently and in line with modern standards.</p>
        <h2>3. Differences Between Reset and Normalize</h2>
        <p>CSS Reset removes all styles, while Normalize.css preserves useful defaults and corrects issues.</p>
        <h2>4. Custom Reset Styles</h2>
        <p>Custom reset styles combine the benefits of both CSS reset and Normalize.css to suit specific needs.</p>
        <h2>5. Browser Default Styles</h2>
        <p>Browser default styles are the built-in styles applied by browsers if no CSS is provided.</p>
    </div>
</body>
</html>
