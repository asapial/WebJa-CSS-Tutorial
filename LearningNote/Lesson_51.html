CSS-in-JS

1. Introduction to CSS-in-JS

Description:
CSS-in-JS refers to a styling approach where CSS is composed using JavaScript. It offers several benefits, such as scoped styles, the ability to leverage JavaScript for dynamic styling, and easier maintenance of styles in component-based architectures.
Key Benefits:
  • Scoped styles prevent conflicts.
  • Dynamic styling based on component state or props.
  • Integration with JavaScript frameworks and libraries.

2. Styled Components

Description:
Styled Components is a popular CSS-in-JS library for React that uses tagged template literals to style components. It allows you to write actual CSS code within JavaScript and bind styles directly to components.
Example: 
import React from 'react';
import styled from 'styled-components';

const Button = styled.button`
  background: ${props => props.primary ? 'blue' : 'gray'};
  color: white;
  font-size: 1em;
  margin: 1em;
  padding: 0.25em 1em;
  border: 2px solid palevioletred;
  border-radius: 3px;
`;

const App = () => (
  <div>
    <Button primary>Primary Button</Button>
    <Button>Secondary Button</Button>
  </div>
);

export default App;

3. Emotion Library

Description:
Emotion is another CSS-in-JS library that provides powerful and flexible styling capabilities. It offers both a styled component API similar to Styled Components and a css function for creating styles.
Example: 
/** @jsxImportSource @emotion/react */
import { css } from '@emotion/react';

const buttonStyle = css`
  background: blue;
  color: white;
  font-size: 1em;
  margin: 1em;
  padding: 0.25em 1em;
  border: 2px solid palevioletred;
  border-radius: 3px;
`;

const App = () => (
  <div>
    <button css={buttonStyle}>Emotion Button</button>
  </div>
);

export default App;

4. CSS Modules

Description:
CSS Modules are a CSS file where all class and animation names are scoped locally by default. They can be used in JavaScript frameworks to ensure that styles are applied only to the component they are intended for, avoiding global scope issues.
Example: 
// styles.module.css
.button {
  background: blue;
  color: white;
  font-size: 1em;
  margin: 1em;
  padding: 0.25em 1em;
  border: 2px solid palevioletred;
  border-radius: 3px;
}

// App.js
import React from 'react';
import styles from './styles.module.css';

const App = () => (
  <div>
    <button className={styles.button}>CSS Module Button</button>
  </div>
);

export default App;

5. Best Practices for CSS-in-JS

Description:
When using CSS-in-JS, it's important to follow best practices to ensure maintainability, performance, and a clean codebase. Here are some best practices:
Best Practices:
  • Keep styles simple: Avoid overcomplicating styles with too much logic.
  • Use theme providers: Centralize styling information like colors and spacing.
  • Optimize performance: Be mindful of potential performance impacts, such as unnecessary re-renders.
  • Leverage static styles: When possible, use static styles to reduce the overhead of dynamic styling.
  • Organize styles logically: Group related styles together and avoid deeply nested components.
Example Combining Best Practices: 
import React from 'react';
import styled, { ThemeProvider } from 'styled-components';

// Define a theme
const theme = {
  primary: 'blue',
  secondary: 'gray',
  borderRadius: '3px',
};

// Define a styled button component
const Button = styled.button`
  background: ${props => props.theme[props.variant]};
  color: white;
  font-size: 1em;
  margin: 1em;
  padding: 0.25em 1em;
  border: 2px solid palevioletred;
  border-radius: ${props => props.theme.borderRadius};
`;

// Define the app component
const App = () => (
  <ThemeProvider theme={theme}>
    <div>
      <Button variant="primary">Primary Button</Button>
      <Button variant="secondary">Secondary Button</Button>
    </div>
  </ThemeProvider>
);

export default App;
