CSS Houdini

1. Introduction to Houdini

Description:
CSS Houdini provides developers with more control over the CSS rendering pipeline. It enables the creation of custom styles and animations that can be efficiently processed by the browser, improving performance and enabling new design possibilities.
Key Features:
  • Access to the rendering pipeline
  • Creation of custom CSS properties
  • Custom paint, layout, and animation effects

2. Paint API

Description:
The Paint API allows developers to create custom paint worklets that can draw directly onto an element's background. These paint worklets can be used to generate complex patterns, gradients, and other visual effects dynamically.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Paint API Example</title>
    <style>
        .paint-box {
            width: 200px;
            height: 200px;
            background-image: paint(myPainter);
        }
    </style>
    <script>
        if (CSS.paintWorklet) {
            CSS.paintWorklet.addModule('paint-worklet.js');
        }
    </script>
</head>
<body>
    <h1>Paint API Example</h1>
    <div class="paint-box"></div>
</body>
</html>

// paint-worklet.js
registerPaint('myPainter', class {
    static get inputProperties() {
        return ['--my-color'];
    }
    paint(ctx, geom, properties) {
        const color = properties.get('--my-color').toString();
        ctx.fillStyle = color;
        ctx.fillRect(0, 0, geom.width, geom.height);
        ctx.beginPath();
        ctx.arc(geom.width / 2, geom.height / 2, geom.width / 4, 0, 2 * Math.PI);
        ctx.fillStyle = 'white';
        ctx.fill();
    }
});

3. Typed OM (Object Model)

Description:
The Typed Object Model (Typed OM) provides a more powerful and type-safe way to interact with CSS values in JavaScript. It replaces the traditional string-based API with a set of typed objects, making it easier to manipulate and perform calculations with CSS properties.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Typed OM Example</title>
    <style>
        .typed-box {
            width: 100px;
            height: 100px;
            background-color: lightblue;
        }
    </style>
</head>
<body>
    <h1>Typed OM Example</h1>
    <div class="typed-box" id="box"></div>
    <script>
        const box = document.getElementById('box');
        const computedStyle = getComputedStyle(box);
        const widthValue = CSSUnitValue.parse(computedStyle.width);
        console.log(widthValue.value, widthValue.unit); // 100, 'px'
        
        const newWidth = widthValue.value + 50;
        box.attributeStyleMap.set('width', CSS.px(newWidth));
    </script>
</body>
</html>

4. Layout API

Description:
The Layout API allows developers to define custom layout algorithms that can be used to lay out elements on the page. This provides a way to create new layout behaviors that are not natively supported by CSS.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Layout API Example</title>
    <style>
        .layout-box {
            display: layout(myCustomLayout);
            width: 300px;
            height: 300px;
            background-color: lightgray;
        }
        .layout-box > div {
            background-color: lightblue;
            border: 1px solid blue;
        }
    </style>
    <script>
        if ('layoutWorklet' in CSS) {
            CSS.layoutWorklet.addModule('layout-worklet.js');
        }
    </script>
</head>
<body>
    <h1>Layout API Example</h1>
    <div class="layout-box">
        <div style="width: 50px; height: 50px;"></div>
        <div style="width: 100px; height: 100px;"></div>
        <div style="width: 75px; height: 75px;"></div>
    </div>
</body>
</html>

// layout-worklet.js
registerLayout('myCustomLayout', class {
    *layout(children, edges, constraints, styleMap) {
        let x = 0, y = 0;
        for (const child of children) {
            child.style.position = 'absolute';
            child.style.left = CSS.px(x);
            child.style.top = CSS.px(y);
            x += child.styleMap.get('width').value;
            y += child.styleMap.get('height').value;
        }
    }
});

5. Houdini and Performance

Description:
Houdini enhances performance by allowing developers to hook into the rendering pipeline. By creating custom paint, layout, and animation effects, you can achieve highly optimized performance for complex visual effects that would be inefficient or impossible with traditional CSS.
Key Points:
  • Reduces the need for JavaScript-driven animations and layouts.
  • Enables efficient, browser-native execution of custom styles.
  • Allows for more creative and performant designs.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Houdini and Performance Example</title>
    <style>
        .performance-box {
            width: 100px;
            height: 100px;
            background-image: paint(performancePainter);
        }
    </style>
    <script>
        if (CSS.paintWorklet) {
            CSS.paintWorklet.addModule('performance-painter.js');
        }
    </script>
</head>
<body>
    <h1>Houdini and Performance Example</h1>
    <div class="performance-box"></div>
</body>
</html>

// performance-painter.js
registerPaint('performancePainter', class {
    paint(ctx, geom) {
        ctx.fillStyle = 'blue';
        ctx.fillRect(0, 0, geom.width, geom.height);
        ctx.fillStyle = 'white';
        ctx.fillText('Houdini', geom.width / 2, geom.height / 2);
    }
});
