Accessibility in CSS

1. Accessible Color Contrast

Accessible color contrast ensures that text is readable for users with visual impairments. The Web Content Accessibility Guidelines (WCAG) recommend a contrast ratio of at least 4.5:1 for normal text and 3:1 for large text.

Key Points:

  • Ensure sufficient contrast between text and background colors.
  • Use tools to check contrast ratios (e.g., WebAIM Contrast Checker).

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Accessible Color Contrast</title>
    <style>
        body {
            font-family: Arial, sans-serif;
        }

        .accessible {
            background-color: #000;
            color: #fff; /* High contrast */
            padding: 10px;
            margin: 10px;
        }

        .inaccessible {
            background-color: #fff;
            color: #ccc; /* Low contrast */
            padding: 10px;
            margin: 10px;
        }
    </style>
</head>
<body>
    <div class="accessible">This text has good contrast.</div>
    <div class="inaccessible">This text has poor contrast.</div>
</body>
</html>

2. Focus States

Focus states highlight interactive elements (e.g., links, buttons) when they are focused, helping users navigate using the keyboard.

Key Points:

  • Ensure focusable elements have visible focus indicators.
  • Use :focus pseudo-class to style focused elements.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Focus States</title>
    <style>
        a, button {
            padding: 10px;
            margin: 10px;
            text-decoration: none;
            color: #3498db;
            border: 2px solid transparent;
        }

        a:focus, button:focus {
            border-color: #3498db;
            outline: none;
        }
    </style>
</head>
<body>
    <a href="#">Focus me</a>
    <button>Focus me</button>
</body>
</html>

3. Skip to Content Links

Skip to content links allow keyboard users to bypass repetitive navigation and jump directly to the main content.

Key Points:

  • Place the skip link at the top of the page.
  • Ensure it is visible when focused.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Skip to Content Link</title>
    <style>
        .skip-link {
            position: absolute;
            top: -40px;
            left: 10px;
            background-color: #3498db;
            color: white;
            padding: 8px;
            z-index: 1000;
        }

        .skip-link:focus {
            top: 10px;
        }

        nav, main {
            margin: 20px;
        }
    </style>
</head>
<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
    <nav>
        <a href="#">Home</a>
        <a href="#">About</a>
        <a href="#">Contact</a>
    </nav>
    <main id="main-content">
        <h1>Main Content</h1>
        <p>This is the main content of the page.</p>
    </main>
</body>
</html>

4. ARIA Roles and Attributes

ARIA (Accessible Rich Internet Applications) roles and attributes enhance the accessibility of web content by providing additional information to assistive technologies.

Key Points:

  • Use roles to define the purpose of an element (e.g., role="button").
  • Use attributes to provide additional information (e.g., aria-label, aria-expanded).

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>ARIA Roles and Attributes</title>
    <style>
        button {
            padding: 10px;
            margin: 10px;
        }
    </style>
</head>
<body>
    <button aria-label="Close" role="button">X</button>
    <nav role="navigation">
        <a href="#" role="link">Home</a>
        <a href="#" role="link">About</a>
        <a href="#" role="link">Contact</a>
    </nav>
</body>
</html>

5. CSS for Screen Readers

CSS can be used to visually hide content while keeping it accessible to screen readers.

Key Points:

  • Use techniques like clip, position: absolute, and width: 1px to hide elements from view but make them available to screen readers.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>CSS for Screen Readers</title>
    <style>
        .sr-only {
            position: absolute;
            width: 1px;
            height: 1px;
            padding: 0;
            margin: -1px;
            overflow: hidden;
            clip: rect(0, 0, 0, 0);
            border: 0;
        }
    </style>
</head>
<body>
    <h1>Web Accessibility</h1>
    <p>This is a demonstration of making content accessible.</p>
    <p class="sr-only">This text is hidden visually but accessible to screen readers.</p>
</body>
</html>

Example Code 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Accessible Web Page</title>
    <style>
        :root {
            --primary-color: #3498db;
            --secondary-color: #2ecc71;
            --background-color: #f0f0f0;
            --text-color: #333;
            --high-contrast-color: #fff;
            --low-contrast-color: #ccc;
            --padding: 10px;
        }

        body {
            background-color: var(--background-color);
            color: var(--text-color);
            font-family: Arial, sans-serif;
            padding: 20px;
        }

        .accessible {
            background-color: var(--primary-color);
            color: var(--high-contrast-color);
            padding: var(--padding);
            margin: 10px;
        }

        .inaccessible {
            background-color: var(--background-color);
            color: var(--low-contrast-color);
            padding: var(--padding);
            margin: 10px;
        }

        a, button {
            padding: var(--padding);
            margin: 10px;
            text-decoration: none;
            color: var(--primary-color);
            border: 2px solid transparent;
        }

        a:focus, button:focus {
            border-color: var(--primary-color);
            outline: none;
        }

        .skip-link {
            position: absolute;
            top: -40px;
            left: 10px;
            background-color: var(--primary-color);
            color: var(--high-contrast-color);
            padding: var(--padding);
            z-index: 1000;
        }

        .skip-link:focus {
            top: 10px;
        }

        .sr-only {
            position: absolute;
            width: 1px;
            height: 1px;
            padding: 0;
            margin: -1px;
            overflow: hidden;
            clip: rect(0, 0, 0, 0);
            border: 0;
        }

        nav, main {
            margin: 20px;
        }
    </style>
</head>
<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
    <nav role="navigation">
        <a href="#" role="link">Home</a>
        <a href="#" role="link">About</a>
        <a href="#" role="link">Contact</a>
    </nav>
    <main id="main-content">
        <h1>Web Accessibility</h1>
        <div class="accessible">This text has good contrast.</div>
        <div class="inaccessible">This text has poor contrast.</div>
        <button aria-label="Close" role="button">X</button>
        <p>This is a demonstration of making content accessible.</p>
        <p class="sr-only">This text is hidden visually but accessible to screen readers.</p>
    </main>
</body>
</html>
