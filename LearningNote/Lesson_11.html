Pseudo-classes and Pseudo-elements

1. :hover

The :hover pseudo-class is triggered when the user hovers over an element with a pointing device (like a mouse). It's commonly used to enhance the user experience by changing the appearance of elements such as links, buttons, and images when the user interacts with them.

Key Points:

  • Activated when the user hovers over an element.
  • Commonly used to change the appearance of interactive elements.
  • Enhances user interaction and feedback.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        a {
            color: blue;
            text-decoration: none;
        }

        a:hover {
            color: red;
            text-decoration: underline;
        }
    </style>
</head>
<body>
    <a href="#">Hover over me!</a>
</body>
</html>

2. :active

The :active pseudo-class applies to an element that is currently being activated by the user. This usually means the moment between when the user presses and releases the mouse button. It provides immediate feedback to indicate that the element has been clicked.

Key Points:

  • Activated during the period when an element is being clicked.
  • Commonly used to provide visual feedback for buttons and links.
  • Indicates to users that their action is being processed.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        button {
            background-color: blue;
            color: white;
            border: none;
            padding: 10px 20px;
            cursor: pointer;
        }

        button:active {
            background-color: red;
        }
    </style>
</head>
<body>
    <button>Click me!</button>
</body>
</html>

3. :focus

The :focus pseudo-class applies to an element when it has received focus, meaning it is ready to accept user input. This is typically used for form elements such as input fields, textareas, and buttons to highlight them when they are active and ready for input.

Key Points:

  • Activated when an element receives focus, such as by clicking or tabbing into it.
  • Commonly used to highlight form elements.
  • Improves accessibility by providing visual cues to users navigating via keyboard.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        input {
            border: 2px solid gray;
            padding: 5px;
        }

        input:focus {
            border-color: blue;
            outline: none;
        }
    </style>
</head>
<body>
    <input type="text" placeholder="Focus on me!">
</body>
</html>

4. ::before

The ::before pseudo-element is used to insert content before an element's actual content. This is often used for adding decorative content or additional information that is not part of the main content flow. The content property is required to specify what is inserted.

Key Points:

  • Inserts content before an element's actual content.
  • Commonly used for decorative purposes or additional information.
  • Requires the content property to define what is inserted.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        .note::before {
            content: "Note: ";
            font-weight: bold;
            color: red;
        }
    </style>
</head>
<body>
    <p class="note">This is a very important message.</p>
</body>
</html>

5. ::after

The ::after pseudo-element is used to insert content after an element's actual content. Like ::before, it is commonly used for decorative purposes or to add additional information. The content property is required to specify what is inserted.

Key Points:

  • Inserts content after an element's actual content.
  • Commonly used for decorative purposes or additional information.
  • Requires the content property to define what is inserted.

Example: 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        .note::after {
            content: " (please read carefully)";
            font-style: italic;
            color: blue;
        }
    </style>
</head>
<body>
    <p class="note">This is a very important message.</p>
</body>
</html>

Example Code 

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <style>
        /* Styling for links */
        a {
            color: blue;
            text-decoration: none;
        }
        a:hover {
            color: red;
            text-decoration: underline;
        }
        a:active {
            color: green;
        }

        /* Styling for buttons */
        button {
            background-color: blue;
            color: white;
            border: none;
            padding: 10px 20px;
            cursor: pointer;
        }
        button:active {
            background-color: red;
        }

        /* Styling for input fields */
        input {
            border: 2px solid gray;
            padding: 5px;
        }
        input:focus {
            border-color: blue;
            outline: none;
        }

        /* Styling for notes with ::before and ::after */
        .note::before {
            content: "Note: ";
            font-weight: bold;
            color: red;
        }
        .note::after {
            content: " (please read carefully)";
            font-style: italic;
            color: blue;
        }
    </style>
</head>
<body>
    <a href="#">Hover and click me!</a>
    <br><br>
    <button>Click me!</button>
    <br><br>
    <input type="text" placeholder="Focus on me!">
    <br><br>
    <p class="note">This is a very important message.</p>
</body>
</html>
