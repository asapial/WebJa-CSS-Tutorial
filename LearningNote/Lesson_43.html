CSS Architecture

1. BEM Methodology

Description:
BEM (Block, Element, Modifier) is a methodology that promotes component-based development by defining clear and consistent naming conventions for classes. This makes the code more readable and maintainable.
  • Block: The main component (e.g., .button).
  • Element: A part of the block (e.g., .button__icon).
  • Modifier: A variation of the block or element (e.g., .button--primary).
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>BEM Methodology Example</title>
    <style>
        .button {
            padding: 10px 20px;
            border: none;
            cursor: pointer;
        }

        .button__icon {
            margin-right: 5px;
        }

        .button--primary {
            background-color: #007BFF;
            color: white;
        }

        .button--secondary {
            background-color: #6C757D;
            color: white;
        }
    </style>
</head>
<body>

2. OOCSS (Object-Oriented CSS)

Description:
OOCSS (Object-Oriented CSS) focuses on separating the structure (HTML) and the skin (CSS). It encourages reusing style patterns and creating small, reusable modules.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>OOCSS Example</title>
    <style>
        .box {
            border: 1px solid #ccc;
            padding: 20px;
            margin: 10px;
        }

        .media {
            display: flex;
            align-items: center;
        }

        .media__image {
            margin-right: 10px;
        }

        .media__content {
            flex: 1;
        }
    </style>
</head>
<body>
    <div class="box media">
        <img src="https://via.placeholder.com/50" alt="Image" class="media__image">
        <div class="media__content">
            <h3>Title</h3>
            <p>Description text goes here.</p>
        </div>
    </div>
</body>
</html>

3. SMACSS (Scalable and Modular Architecture for CSS)

Description:
SMACSS (Scalable and Modular Architecture for CSS) is a style guide for writing scalable and maintainable CSS. It categorizes CSS rules into five types: Base, Layout, Module, State, and Theme.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>SMACSS Example</title>
    <style>
        /* Base */
        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
        }

        /* Layout */
        .layout-header {
            background-color: #007BFF;
            color: white;
            padding: 10px 20px;
        }

        .layout-footer {
            background-color: #333;
            color: white;
            padding: 10px 20px;
            text-align: center;
        }

        /* Module */
        .card {
            border: 1px solid #ccc;
            padding: 20px;
            margin: 10px;
        }

        /* State */
        .is-active {
            background-color: #f0f0f0;
        }

        /* Theme */
        .theme-dark .card {
            background-color: #333;
            color: white;
        }
    </style>
</head>
<body>
    <header class="layout-header">
        <h1>Header</h1>
    </header>
    <div class="card is-active">
        <h2>Card Title</h2>
        <p>Card content goes here.</p>
    </div>
    <footer class="layout-footer">
        Footer
    </footer>
</body>
</html>

4. ITCSS (Inverted Triangle CSS)

Description:
ITCSS (Inverted Triangle CSS) is a methodology for organizing CSS in a way that is scalable and maintainable. It organizes CSS from generic to specific, starting with global settings and moving to components.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>ITCSS Example</title>
    <style>
        /* Settings */
        :root {
            --primary-color: #007BFF;
        }

        /* Tools */
        .clearfix::after {
            content: "";
            display: block;
            clear: both;
        }

        /* Generic */
        * {
            box-sizing: border-box;
        }

        body {
            margin: 0;
            font-family: Arial, sans-serif;
        }

        /* Elements */
        a {
            color: var(--primary-color);
        }

        /* Objects */
        .media {
            display: flex;
            align-items: center;
        }

        .media__image {
            margin-right: 10px;
        }

        /* Components */
        .button {
            padding: 10px 20px;
            border: none;
            cursor: pointer;
            background-color: var(--primary-color);
            color: white;
        }

        /* Utilities */
        .u-mt-20 {
            margin-top: 20px;
        }
    </style>
</head>
<body>
    <div class="media u-mt-20">
        <img src="https://via.placeholder.com/50" alt="Image" class="media__image">
        <div>
            <h3>Title</h3>
            <p>Description text goes here.</p>
            <a href="#" class="button">Learn More</a>
        </div>
    </div>
</body>
</html>

5. Atomic CSS

Description:
Atomic CSS is a methodology that uses small, single-purpose classes to apply styles. Each class corresponds to a single CSS property. This promotes reusability and reduces redundancy.
Example: 
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Atomic CSS Example</title>
    <style>
        .p-10 {
            padding: 10px;
        }

        .m-10 {
            margin: 10px;
        }

        .bg-primary {
            background-color: #007BFF;
        }

        .text-white {
            color: white;
        }

        .d-flex {
            display: flex;
        }

        .align-center {
            align-items: center;
        }

        .mr-10 {
            margin-right: 10px;
        }
    </style>
</head>
<body>
    <div class="d-flex align-center m-10 p-10 bg-primary">
        <img src="https://via.placeholder.com/50" alt="Image" class="mr-10">
        <div class="text-white">
            <h3>Title</h3>
            <p>Description text goes here.</p>
        </div>
    </div>
</body>
</html>

    <script>
        const toggleButton = document.querySelector('.theme-toggle');
        toggleButton.addEventListener('click', () => {
            if (document.body.classList.contains('dark-theme')) {
                document.body.classList.remove('dark-theme');
                document.body.classList.add('light-theme');
            } else {
                document.body.classList.remove('light-theme');
                document.body.classList.add('dark-theme');
            }
        });

        // Apply user preference for dark mode
        if (window.matchMedia && window.matchMedia('(prefers-color-scheme: dark)').matches) {
            document.body.classList.add('dark-theme');
        } else {
            document.body.classList.add('light-theme');
        }
    </script>
</body>
</html>

Sass (styles.scss) 

// Define theme variables
$background-color-light: #ffffff;
$text-color-light: #000000;
$background-color-dark: #333333;
$text-color-dark: #ffffff;

// Mixin for theme
@mixin theme($bg-color, $text-color) {
    background-color: $bg-color;
    color: $text-color;
}

// Apply themes
body.light-theme {
    @include theme($background-color-light, $text-color-light);
}

body.dark-theme {
    @include theme($background-color-dark, $text-color-dark);
}

// User preference media query for dark mode
@media (prefers-color-scheme: dark) {
    body {
        background-color: $background-color-dark;
        color: $text-color-dark;
    }
}
